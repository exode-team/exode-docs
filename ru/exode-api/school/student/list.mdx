## Параметры запроса

Эндпоинт возвращает **постраничный** список учеников школы вместе с их Telegram ID и доступами к продуктам.

<Info>
  **Ученик** — это пользователь, у которого есть доступ к продукту школы. Возвращаются **уникальные** ученики: один человек = одна строка, даже если у него несколько курсов.
</Info>

### Фильтры

<ParamField query="active" type="boolean" required={false}>
  Фильтр по статусу доступа. `true` — только ученики с активным доступом, `false` — только с неактивным доступом.
</ParamField>

<ParamField query="search" type="string" required={false}>
  Поиск по email, телефону или имени ученика.
</ParamField>

### Пагинация

<ParamField query="take" type="integer" required={false}>
  Количество записей на странице. Максимум `1000`, по умолчанию `100`.
</ParamField>

<ParamField query="page" type="integer" required={false}>
  Номер страницы, начиная с `1`.
</ParamField>

<ParamField query="skip" type="integer" required={false}>
  Количество пропускаемых записей. Альтернатива параметру `page`.
</ParamField>

```javascript Node.js
  try {
```

```php PHP

  $result = json_decode($response, true);

  echo $response;
}
?>
```

```python Python
import requests
```

## Описание полей

<ResponseField name="userId" type="integer">
  ID пользователя в системе exode.
</ResponseField>

<ResponseField name="extId" type="string | null">
  Внешний идентификатор пользователя из вашей системы.
</ResponseField>

<ResponseField name="tgId" type="integer | null">
  Telegram ID пользователя. Нужен для рассылки в Telegram.
</ResponseField>

<ResponseField name="login" type="string">
  Логин пользователя (email, телефон или домен).
</ResponseField>

<ResponseField name="email" type="string | null">
  Email адрес ученика.
</ResponseField>

<ResponseField name="phone" type="string | null">
  Номер телефона ученика.
</ResponseField>

<ResponseField name="name" type="string">
  Имя ученика.
</ResponseField>

<ResponseField name="accesses" type="object[]">
  Массив доступов ученика (продукт / курс).

  <Expandable title="Свойства accesses">
    <ResponseField name="productId" type="integer">
      ID продукта.
    </ResponseField>

    <ResponseField name="courseId" type="integer | null">
      ID курса.
    </ResponseField>

    <ResponseField name="active" type="boolean">
      Активен ли доступ.
    </ResponseField>

    <ResponseField name="expireAt" type="string | null">
      Дата окончания доступа в формате ISO. `null`, если доступ бессрочный.
    </ResponseField>
  </Expandable>
</ResponseField>

## Требования к правам доступа

<Check>
Для получения списка учеников требуется **право на управление пользователями школы** (`SchoolManageUsers`) или **право на управление учениками курса** (`CourseStudentManage`).
</Check>

<Warning>
  Сервисный пользователь должен быть аутентифицирован по токену и иметь соответствующие права доступа к указанной
  школе.
</Warning>

<Info>
  Возвращаются только ученики в рамках указанной школы.
</Info>
