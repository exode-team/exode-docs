```
POST /saas/v2/user/create
```

Требуется аутентификация и право `SchoolManageUsers`.

## Параметры запроса

### Поля для входа (логин)

Логином для входа в школу являются поля **`email`, `phone` и `domain`** — для автоматической отправки данных входа
передайте хотя бы одно из них.

<Warning>
  `extId` и `tgId` **не являются логином** — авторизоваться по ним нельзя. Они используются только как
  идентификаторы: `extId` — для связи с вашей CRM/LMS, `tgId` — для связи аккаунта с Telegram.
</Warning>

<Info>
  Логин и пароль для входа отправляется **автоматически**.
  Если не передать одно из полей ниже — то вам необходимо будет получить пароль и отправить его самостоятельно в
  настройках пользователя
</Info>

<Info>
  Если передать поле `password` — пользователю будет установлен указанный пароль, а автоматическая генерация и
  отправка пароля выполняться **не будет**.
</Info>

<Info>
  Для отправки логина и пароля в Telegram — пользователь должен дать разрешение писать боту или иметь активный чат с
  этим ботом
</Info>

<ParamField body="email" type="string" required={false}>
  Email адрес пользователя. Должен быть валидным email форматом. При передаче пустой строки — преобразуется в
  `null`.
</ParamField>

<ParamField body="phone" type="string" required={false}>
  Номер телефона пользователя. Должен быть в международном формате (например, +9876543210). При передаче пустой строки — преобразуется в `null`.
</ParamField>

<ParamField body="domain" type="string" required={false}>
  Доменный логин пользователя. До 65 символов: латинские буквы, цифры, `_` и точки — точка не может быть первой/последней и не может
  идти подряд (автоматически приводится к нижнему
  регистру). Должен быть уникальным в рамках школы. Чисто числовые значения и значения вида `id12345`
  зарезервированы системой — задать их вручную нельзя (вернётся `DomainIsBusy`). Если не передан — генерируется
  автоматически в формате `id12345`. Удобен, когда логины формируются на вашей стороне и у пользователя нет email
  или телефона.
</ParamField>

### Идентификаторы для интеграции

<ParamField body="extId" type="string" required={false}>
  Внешний идентификатор пользователя из вашей системы (например, GUID из CRM/1С). Строка до 50 символов
  без `/` и пробелов (используется в пути `user/ext/{extId}/update`) или `null`. При передаче пустой строки —
  преобразуется в `null`. Используется для связи и поиска пользователя (`user/find`, `user/upsert`),
  **для входа не используется**.
</ParamField>

<ParamField body="tgId" type="integer" required={false}>
  Telegram ID пользователя. Целое число или `null`.
</ParamField>

### Дополнительные параметры

<ParamField body="status" type="enum" required={false}>
  Статус учётной записи: `Active`, `OnLeave`, `Banned`, `Blocked` или `Terminated`. По умолчанию `Active`.
  - `Active` — обычный доступ;
  - `OnLeave` — «в отсутствии» (отпуск, больничный и т.п.): информационный статус, вход и доступ **не блокирует**.
    Управляется автоматически модулем [отсутствий](/ru/exode-api/school/staff/absence);
  - `Banned` — забанен: вход запрещён, активные сессии завершаются. При снятии бана фактический статус
    пересчитывается автоматически (по трудоустройствам и отсутствиям);
  - `Blocked` — заблокирован администратором: вход запрещён, активные сессии завершаются, зачисления на продукты
    блокируются. Пользователь остаётся видимым в отчётах; возврат в `Active` восстанавливает доступ;
  - `Terminated` — уволен: доступ закрыт так же, как у `Blocked`. Ставится автоматически при увольнении с последнего
    активного трудоустройства; повторный найм возвращает `Active`.

  Статус `Deleted` зарезервирован системой (удаление аккаунта) — передать его нельзя.
</ParamField>

<Info>
  Булево поле `banned` из запроса **удалено** — блокировка управляется только через `status` (`Banned` /
  `Blocked`). В ответе поля `banned` и `active` по-прежнему присутствуют как производные от `status`.
</Info>

<ParamField body="password" type="string" required={false}>
  Пароль пользователя. От 6 до 100 символов. Если передан — устанавливается как пароль для входа, автоматическая
  отправка сгенерированного пароля пропускается. При передаче пустой строки — преобразуется в `null`.
  Доступно только при создании (в `update` поле не поддерживается).
</ParamField>

### Параметры профиля

<ParamField body="profile" type="object" required={false}>
  Объект с данными профиля пользователя.

  <Expandable title="Свойства профиля">
    <ParamField body="firstName" type="string" required={false}>
      Имя пользователя. Максимум 15 символов, только буквы (латиница и кириллица) и пробелы. Автоматически
      обрезаются пробелы в начале и конце.
    </ParamField>

    <ParamField body="lastName" type="string" required={false}>
      Фамилия пользователя. Максимум 15 символов, только буквы (латиница и кириллица) и пробелы. Автоматически
      обрезаются пробелы в начале и конце.
    </ParamField>

    <ParamField body="bdate" type="string" required={false}>
      Дата рождения в формате YYYY-MM-DD.
    </ParamField>

    <ParamField body="sex" type="enum" required={false}>
      Пол пользователя. Возможные значения: `Ufo`, `Women`, `Men`.
    </ParamField>

    <ParamField body="role" type="enum" required={false}>
      Роль пользователя. Возможные значения: `Student`, `Tutor`, `Parent`.
    </ParamField>
  </Expandable>
</ParamField>

<Info>
  При создании пользователя автоматически создается связанный профиль с указанными данными. Если профиль не указан,
  создается пустой профиль, поля `firstName` и `lastName` будут заполнены пользователем при первом входе в аккаунт.
</Info>

### Дополнительные агрегаты (`extra`)

<ParamField body="extra" type="object" required={false}>
  Контейнер связанных данных, применяемых вместе с созданием пользователя. Сейчас содержит блок `staff`
  (трудоустройства); в будущем будет расширяться другими агрегатами.

  <Expandable title="Свойства extra">
    <ParamField body="staff" type="object" required={false}>
      Данные о трудоустройстве создаваемого сотрудника. Доступно **только для корпоративных школ**, для них блок
      **обязателен** — сотрудник не может существовать без трудоустройства.

      <Expandable title="Свойства staff">
        <ParamField body="employments" type="array" required>
          Массив трудоустройств (от 1 до 10). Несколько элементов — это совместительство: например, основное место
          работы + внутреннее совместительство в другом департаменте.

          Должность в каждом элементе указывается через `positionId` **или** `positionExtId`, департамент — через
          `departmentId` **или** `departmentExtId` (внешние идентификаторы из вашей системы, см. разделы
          [Должности](/ru/exode-api/school/staff/position) и [Департаменты](/ru/exode-api/school/staff/department)).

          <Expandable title="Свойства элемента employments">
            <ParamField body="extId" type="string" required={false}>
              Внешний идентификатор трудоустройства из вашей системы. От 1 до 50 символов, без `/` и пробелов.
              Уникален в рамках школы среди открытых трудоустройств.
            </ParamField>

            <ParamField body="positionId" type="integer" required={false}>
              ID должности в Exode. Обязателен, если не передан `positionExtId`.
            </ParamField>

            <ParamField body="positionExtId" type="string" required={false}>
              Внешний идентификатор должности из вашей системы. От 1 до 50 символов. Обязателен, если не передан
              `positionId`.
            </ParamField>

            <ParamField body="departmentId" type="integer" required={false}>
              ID департамента в Exode. Обязателен, если не передан `departmentExtId`.
            </ParamField>

            <ParamField body="departmentExtId" type="string" required={false}>
              Внешний идентификатор департамента из вашей системы. От 1 до 50 символов. Обязателен, если не передан
              `departmentId`.
            </ParamField>

            <ParamField body="kind" type="enum" required={false}>
              Вид занятости: `Main` (основное место работы), `InternalSecondary` (внутреннее совместительство),
              `ExternalSecondary` (внешнее совместительство). По умолчанию `Main`.
            </ParamField>

            <ParamField body="type" type="enum" required={false}>
              Тип занятости: `FullTime` (полная) или `PartTime` (частичная). По умолчанию `FullTime`.
            </ParamField>

            <ParamField body="startAt" type="string" required={false}>
              Дата начала трудоустройства в формате ISO 8601. По умолчанию — текущий момент.
            </ParamField>

            <ParamField body="rate" type="number" required={false}>
              Ставка — доля полной ставки, от `0.01` до `1`. По умолчанию `1`.
            </ParamField>
          </Expandable>
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

<Info>
  Данные, которых нет в этой схеме (город, произвольный статус из CRM и т.п.), передаются через
  [кастомные поля](/ru/exode-api/school/custom-field/set). Управление департаментами, должностями и
  трудоустройствами после создания — в разделе [Сотрудники (HR)](/ru/exode-api/school/staff/employment).
</Info>

<?php

$url = 'https://api.exode.biz/saas/v2/user/create';
$data = [
  'email' => 'user@example.com',
  'phone' => '+9876543210',
  'domain' => 'i_ivanov',
  'extId' => 'crm_12345',
  'tgId' => null,

  'School-Id: {{ schoolId }}',
  'Content-Type: application/json',
  'Authorization: Bearer YOUR_TOKEN'
];

$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, $url);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, $headers);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
$httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
curl_close($ch);

if ($httpCode === 200) {
  $result = json_decode($response, true);
  echo "User created successfully\n";
  print_r($result['payload']);
} else {
  echo "Error: HTTP $httpCode\n";
  echo $response;
}

import json
