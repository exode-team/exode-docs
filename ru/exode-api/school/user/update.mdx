```
PUT /saas/v2/user/:userId/update
```

Требуется аутентификация и право `SchoolManageUsers`.

## Параметры запроса

##### При обновлении все поля необязательны для заполнения. Если поле необходимо удалить и оно может быть пустым — передавайте `null`

<ParamField body="email" type="string" required={false}>
  Email адрес пользователя. Должен быть валидным email форматом. При передаче пустой строки — преобразуется в
  `null`.
</ParamField>

<ParamField body="phone" type="string" required={false}>
  Номер телефона пользователя. Должен быть в международном формате (например, +9876543210). При передаче пустой строки — преобразуется в `null`.
</ParamField>

<ParamField body="domain" type="string" required={false}>
  Доменный логин пользователя. До 65 символов: латинские буквы, цифры, `_` и точки — точка не может быть первой/последней и не может
  идти подряд (автоматически приводится к нижнему
  регистру). Должен быть уникальным в рамках школы. Используется для входа наряду с `email` и `phone`.
</ParamField>

<ParamField body="tgId" type="integer" required={false}>
  Telegram ID пользователя. Целое число или `null`. Логином не является — используется для связи аккаунта с Telegram.
</ParamField>

<ParamField body="extId" type="string" required={false}>
  Внешний идентификатор пользователя из вашей системы. Строка до 50 символов или `null` (пустая строка сбрасывает
  значение в `null`). Логином не является — используется только для связи с вашей CRM/LMS и поиска пользователя.
</ParamField>

<ParamField body="banned" type="boolean" required={false}>
  Статус блокировки пользователя. При установке `true` все активные сессии пользователя будут завершены.
</ParamField>

<ParamField body="status" type="enum" required={false}>
  Статус учётной записи: `Active`, `OnLeave`, `Blocked` или `Terminated`.
  - `Active` — обычный доступ;
  - `OnLeave` — «в отсутствии»: информационный статус, доступ **не блокирует**. Управляется автоматически модулем
    [отсутствий](/ru/exode-api/school/staff/absence) — вручную устанавливать не рекомендуется;
  - `Blocked` — заблокирован: вход запрещён, активные сессии завершаются, зачисления на продукты блокируются.
    Возврат в `Active` восстанавливает доступ;
  - `Terminated` — уволен: доступ закрыт так же, как у `Blocked`. Управляется автоматически модулем
    [трудоустройств](/ru/exode-api/school/staff/employment) (увольнение с последнего активного трудоустройства);
    при увольнении используйте `staff/employment/terminate`, а не прямую установку статуса.
</ParamField>

<Warning>
  Владелец школы защищён: другой администратор не может изменить его данные, а перевод владельца в `banned: true`
  или блокирующий статус (`Blocked`/`Terminated`) запрещён даже для него самого — вернётся ошибка
  `ForbiddenModifySchoolOwner`.
</Warning>

<ParamField body="extra" type="object" required={false}>
  Контейнер связанных данных — как в
  [`user/create`](/ru/exode-api/school/user/create#дополнительные-агрегаты-extra): блок `staff.employments`
  (массив 1–10). Только для корпоративных школ; в отличие от создания — **необязателен**. Трудоустройства
  применяются идемпотентно: повторная передача той же активной пары департамент + должность не создаёт дубль.
</ParamField>

<Info>
  Через `extra` в `update` можно только **донанять** сотрудника. Переводы, смена должности и увольнение —
  через эндпоинты раздела [Сотрудники (HR)](/ru/exode-api/school/staff/employment).
</Info>

### Параметры профиля

<ParamField body="profile" type="object" required={false}>
  Объект с данными профиля пользователя для обновления.

  <Expandable title="Свойства профиля">
    <ParamField body="firstName" type="string" required={false}>
      Имя пользователя. Максимум 15 символов, только буквы (латиница и кириллица) и пробелы. Автоматически
      обрезаются пробелы в начале и конце.
    </ParamField>

    <ParamField body="lastName" type="string" required={false}>
      Фамилия пользователя. Максимум 15 символов, только буквы (латиница и кириллица) и пробелы. Автоматически
      обрезаются пробелы в начале и конце.
    </ParamField>

    <ParamField body="bdate" type="string" required={false}>
      Дата рождения в формате YYYY-MM-DD.
    </ParamField>

    <ParamField body="sex" type="enum" required={false}>
      Пол пользователя. Возможные значения: `Ufo`, `Women`, `Men`.
    </ParamField>

    <ParamField body="role" type="enum" required={false}>
      Роль пользователя. Возможные значения: `Student`, `Tutor`, `Parent`.
    </ParamField>
  </Expandable>
</ParamField>

<Info>
  При обновлении пользователя можно изменить как основные данные, так и данные профиля. Если профиль не указан,
  то соответственно, он остается без изменений.
</Info>

## Обновление пользователя по внешнему идентификатору

```
PUT /saas/v2/user/ext/{extId}/update
```

Требуется аутентификация и право `SchoolManageUsers`.

<Info>
  То же самое, что и обновление по `userId`, но пользователь адресуется по внешнему идентификатору `extId`
  в рамках школы — удобно для интеграций (CRM/1С), где внутренние id Exode не хранятся. Тело запроса, ответ,
  защита владельца школы и поддержка `extra` — идентичны обычному `update`.
</Info>

### Параметры

<ParamField path="extId" type="string" required>
  Внешний идентификатор пользователя в рамках школы. Не может содержать `/` и пробельные символы;
  не-ASCII значения (кириллица и т.п.) в пути должны быть **URL-encoded** (`encodeURIComponent`).
</ParamField>

<RequestExample>
```bash cURL
curl --location --request PUT 'https://api.exode.biz/saas/v2/user/ext/i.ivanov.01011990/update' \
  --header 'Seller-Id: {{ sellerId }}' \
  --header 'School-Id: {{ schoolId }}' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer YOUR_TOKEN' \
  --data-raw '{
    "email": "updated@example.com",
    "profile": {
      "firstName": "UpdatedFirstname"
    }
  }'
```

```javascript Node.js
const axios = require('axios');

const updateUserByExtId = async (extId) => {
  const { data } = await axios.put(
    `https://api.exode.biz/saas/v2/user/ext/${encodeURIComponent(extId)}/update`,
    {
      email: 'updated@example.com',
      profile: {
        firstName: 'UpdatedFirstname',
      },
    },
    {
      headers: {
        'Seller-Id': '{{ sellerId }}',
        'School-Id': '{{ schoolId }}',
        'Content-Type': 'application/json',
        'Authorization': 'Bearer YOUR_TOKEN',
      },
    }
  );
  console.log(data.payload.user);
};

updateUserByExtId('i.ivanov.01011990');
```

## Требования к правам доступа

<Check>
Для обновления пользователя требуется **право на управление пользователями школы**.
</Check>

<Warning>
  Сервисный пользователь должен быть аутентифицирован по токену и иметь соответствующие права доступа к указанной
  школе.
</Warning>

<Info>
  При установке статуса `banned: true` все активные сессии пользователя **автоматически завершаются** — это
  реализовано для обеспечения безопасности.
</Info>
