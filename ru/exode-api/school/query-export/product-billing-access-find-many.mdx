**type:** `QUERY_EXPORT_TYPE_PRODUCT_BILLING_ACCESS_FIND_MANY`

Выгружает биллинговые записи доступов к продуктам: статусы списаний (подписки/рассрочки), даты старта,
завершения и повторных попыток списания.

Это тип отчёта для метода [`query-export/generate`](/ru/exode-api/school/query-export/generate): передайте его
в поле `type`, а описанные ниже поля — в `variables`.

<Info>
  Отчёт строится по доступам к продуктам (`productAccessManageFindMany`) с акцентом на биллинг (подписки и
  рассрочки). `variables` — фильтр и сортировка доступов (`FilterAccessProductInput` / `SortAccessProductInput`),
  та же структура, что у отчёта [«Студенты школы»](/ru/exode-api/school/query-export/school-student-find-many).
  Неизвестные поля сервис экспорта игнорирует.
</Info>

---

### Переменные (`variables`)

<ParamField body="filter" type="object">
  Параметры фильтрации доступов (биллинговые поля — с префиксом `billing`).

  <Expandable title="Свойства filter">
    <ParamField body="userIds" type="integer[]">Фильтр по ID пользователей.</ParamField>
    <ParamField body="accessIds" type="integer[]">Фильтр по ID доступов.</ParamField>
    <ParamField body="active" type="boolean">Только активные (`true`) или деактивированные (`false`) доступы.</ParamField>
    <ParamField body="billingActive" type="boolean">Только доступы с активным биллингом (подписка/рассрочка).</ParamField>
    <ParamField body="hasProductBillingTypes" type="string[]">
      Тип биллинга. Массив из значений: `Subscription`, `Installment`.
    </ParamField>
    <ParamField body="billingStatuses" type="string[]">
      Статус биллинга. Массив из значений: `Scheduled`, `Processing`, `WaitingEntryFee`, `Paid`, `Failed`,
      `CanceledByUser`, `CanceledByFailedAutoCharge`, `CanceledByFailedManualCharge`,
      `CanceledByPreviousCancellation`, `CanceledByReinitialization`.
    </ParamField>
    <ParamField body="billingIntervals" type="string[]">
      Интервал списания. Массив из значений: `Week`, `Month`, `Year`.
    </ParamField>
    <ParamField body="billingInvoiceIds" type="integer[]">Фильтр по ID счетов биллинга.</ParamField>
    <ParamField body="billingAmountRange" type="object">
      Диапазон суммы списания. Свойства `from` и `to` — числа.
    </ParamField>
    <ParamField body="billingCurrentPaymentAtDateRange" type="object">
      Диапазон дат последнего списания. Свойства `from` и `to` — даты ISO 8601.
    </ParamField>
    <ParamField body="billingNextPaymentAtDateRange" type="object">
      Диапазон дат следующего списания. Свойства `from` и `to` — даты ISO 8601.
    </ParamField>
    <ParamField body="product" type="object">Вложенный фильтр по продукту (`productIds` и т.д.).</ParamField>
    <ParamField body="price" type="object">Вложенный фильтр по тарифу.</ParamField>
  </Expandable>
</ParamField>

<ParamField body="sort" type="object">
  Параметры сортировки. Значения: `ASC` (по возрастанию) или `DESC` (по убыванию).

  <Expandable title="Свойства sort">
    <ParamField body="id" type="string">Сортировка по ID. `ASC` | `DESC`.</ParamField>
    <ParamField body="createdAt" type="string">Сортировка по дате выдачи доступа. `ASC` | `DESC`.</ParamField>
    <ParamField body="updatedAt" type="string">Сортировка по дате обновления. `ASC` | `DESC`.</ParamField>
    <ParamField body="billingStatus" type="string">Сортировка по статусу биллинга. `ASC` | `DESC`.</ParamField>
    <ParamField body="billingPercent" type="string">Сортировка по проценту оплаты. `ASC` | `DESC`.</ParamField>
    <ParamField body="currentPaymentAt" type="string">Сортировка по дате последнего списания. `ASC` | `DESC`.</ParamField>
    <ParamField body="nextPaymentAt" type="string">Сортировка по дате следующего списания. `ASC` | `DESC`.</ParamField>
    <ParamField body="billingPaidCount" type="string">Сортировка по числу выполненных списаний. `ASC` | `DESC`.</ParamField>
  </Expandable>
</ParamField>

### Колонки отчёта

Лист **Billing Access**: ID пользователя, Фамилия Имя, Способ входа, Номер телефона, Почта, Telegram, ID платежа,
Дата последнего платежа, Дата следующего платежа, Оплачено на последнем платеже, Внесенная сумма, Общая сумма,
_Процент платежей_, _Кол-во выполненных платежей_, _Количество платежей_, _Кол-во продлений_, _Период продления_,
ID курса, Название курса, ID тарифа, Тариф, Способ оплаты, Статус.

<Info>
    Колонки _Процент платежей_, _Кол-во выполненных платежей_ и _Количество платежей_ появляются только при
    фильтре по рассрочке (`hasProductBillingTypes: ["Installment"]`), а _Кол-во продлений_ и _Период продления_ —
    только при фильтре по подписке (`hasProductBillingTypes: ["Subscription"]`).
</Info>

Дополнительный лист **Billing Details** — построчная разбивка платежей: ID пользователя, ID доступа, ID биллинга,
Номер, Дата платежа, Статус, Сумма, ID инвойса, Использованные попытки, Всего попыток, Дата следующей попытки,
Дата отмены.
