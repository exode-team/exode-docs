<Info>
    Руководитель департамента привязывается к **активному трудоустройству** сотрудника. У департамента может быть
    несколько руководителей, но только один из них — основной (`isPrimary`).
</Info>

<Warning>
    Все эндпоинты модуля staff доступны **только** для школ сегмента `Corporate`. Для остальных сегментов запрос
    вернёт ошибку `403 Forbidden`.
</Warning>

## Назначить руководителя

```
POST /saas/v2/staff/department-manager/set
```

Требуется аутентификация и право `StaffManage`.

Эндпоинт работает как upsert: если руководитель для пары департамент + трудоустройство уже назначен —
обновляются флаг `isPrimary` и `extId`, иначе создаётся новая запись. У департамента может быть только один
основной руководитель: при `isPrimary=true` предыдущий основной руководитель департамента автоматически
понижается (флаг `isPrimary` у него сбрасывается в `false`). Департамент и трудоустройство должны принадлежать
школе, трудоустройство должно быть активным — иначе вернутся ошибки `StaffDepartmentNotFound` /
`StaffEmploymentNotFound`.

Департамент и трудоустройство можно указывать как по числовому ID, так и по внешнему идентификатору:
в каждой из пар `departmentId`/`departmentExtId` и `employmentId`/`employmentExtId` передаётся ровно одно поле.

### Параметры запроса

<ParamField body="departmentId" type="integer" required={false}>
    ID департамента. Департамент должен принадлежать школе. Обязателен, если не передан `departmentExtId`.
</ParamField>

<ParamField body="departmentExtId" type="string" required={false}>
    Внешний идентификатор департамента (`extId`). Альтернатива `departmentId` — передаётся ровно одно поле из пары.
</ParamField>

<ParamField body="employmentId" type="integer" required={false}>
    ID активного трудоустройства сотрудника, который назначается руководителем. Обязателен, если не передан
    `employmentExtId`.
</ParamField>

<ParamField body="employmentExtId" type="string" required={false}>
    Внешний идентификатор трудоустройства (`extId`). Альтернатива `employmentId` — передаётся ровно одно поле
    из пары.
</ParamField>

<ParamField body="extId" type="string" required={false}>
    Внешний идентификатор записи руководителя из системы клиента (CRM/1C). От 1 до 50 символов, не может содержать
    `/` и пробельные символы — значение используется в путях `ext/{extId}`. Должен быть уникальным в рамках школы
    среди неудалённых записей, иначе вернётся ошибка `StaffDepartmentManagerExtIdIsNotUniq`.
</ParamField>

<ParamField body="isPrimary" type="boolean" required={false}>
    Назначить сотрудника основным руководителем департамента. По умолчанию `false`. При `true` предыдущий основной
    руководитель автоматически перестаёт быть основным.
</ParamField>

<RequestExample>
```bash cURL
curl --location 'https://api.exode.biz/saas/v2/staff/department-manager/set' \
  --header 'Seller-Id: {{ sellerId }}' \
  --header 'School-Id: {{ schoolId }}' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer YOUR_TOKEN' \
  --data-raw '{
    "departmentId": 4,
    "employmentId": 9,
    "extId": "dm-4b6a-9c2e",
    "isPrimary": true
  }'
```

```javascript Node.js
const axios = require('axios');

const setManager = async () => {

    extId: 'dm-4b6a-9c2e',
    isPrimary: true,
  }, {
    headers: {
      'Seller-Id': '{{ sellerId }}',
      'School-Id': '{{ schoolId }}',
      'Content-Type': 'application/json',
      'Authorization': 'Bearer YOUR_TOKEN',
    },
  });

setManager();
```

## Снять руководителя

```
DELETE /saas/v2/staff/department-manager/{managerId}/remove
```

Требуется аутентификация и право `StaffManage`.

Удаляет запись руководителя департамента (мягкое удаление). После снятия основного руководителя департамент
остаётся без основного руководителя, пока не будет назначен новый.

### Параметры запроса

<ParamField path="managerId" type="integer" required>
    ID записи руководителя департамента (поле `id` из ответа на назначение).
</ParamField>

### Ответ

<ResponseField name="affected" type="integer" required>
    Количество удалённых записей (`1` при успешном снятии руководителя).
</ResponseField>

<RequestExample>
```bash cURL
curl --location --request DELETE 'https://api.exode.biz/saas/v2/staff/department-manager/2/remove' \
  --header 'Seller-Id: {{ sellerId }}' \
  --header 'School-Id: {{ schoolId }}' \
  --header 'Authorization: Bearer YOUR_TOKEN'
```

```javascript Node.js
const axios = require('axios');

const removeManager = async () => {
  const { data } = await axios.delete('https://api.exode.biz/saas/v2/staff/department-manager/2/remove', {
    headers: {
      'Seller-Id': '{{ sellerId }}',
      'School-Id': '{{ schoolId }}',
      'Authorization': 'Bearer YOUR_TOKEN',
    },
  });

  console.log(data.payload);
};

removeManager();
```
</RequestExample>

## Снять руководителя по extId

```
DELETE /saas/v2/staff/department-manager/ext/{extId}/remove
```

Требуется аутентификация и право `StaffManage`.

Аналог снятия по `managerId`, но запись руководителя ищется по внешнему идентификатору (`extId`) в рамках школы.
Удаляет запись руководителя департамента (мягкое удаление). Если запись с указанным `extId` не найдена среди
неудалённых записей школы — вернётся ошибка `StaffDepartmentManagerNotFound`.

### Параметры запроса

<ParamField path="extId" type="string" required>
    Внешний идентификатор записи руководителя департамента (`extId`). Значение в пути должно быть URL-encoded.
</ParamField>

### Ответ

<ResponseField name="affected" type="integer" required>
    Количество удалённых записей (`1` при успешном снятии руководителя).
</ResponseField>

<RequestExample>
```bash cURL
curl --location --request DELETE 'https://api.exode.biz/saas/v2/staff/department-manager/ext/dm-4b6a-9c2e/remove' \
  --header 'Seller-Id: {{ sellerId }}' \
  --header 'School-Id: {{ schoolId }}' \
  --header 'Authorization: Bearer YOUR_TOKEN'
```

```javascript Node.js
const axios = require('axios');

const removeManagerByExtId = async () => {
  const extId = encodeURIComponent('dm-4b6a-9c2e');

  const { data } = await axios.delete(`https://api.exode.biz/saas/v2/staff/department-manager/ext/${extId}/remove`, {
    headers: {
      'Seller-Id': '{{ sellerId }}',
      'School-Id': '{{ schoolId }}',
      'Authorization': 'Bearer YOUR_TOKEN',
    },
  });

  console.log(data.payload);
};

removeManagerByExtId();
```
</RequestExample>

## Требования к правам доступа

<Check>
    Модуль staff доступен только для школ сегмента `Corporate`. Для назначения и снятия руководителей департаментов
    требуется право `StaffManage`.
</Check>

<Warning>
    Сервисный пользователь должен быть аутентифицирован по токену и иметь соответствующие права доступа к указанной
    школе.
</Warning>
