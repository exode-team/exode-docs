<Info>
    Отсутствие (`absence`) фиксирует период, когда сотрудник не работает: отсутствие без уточнения причины
    (`Absent`), отпуск (`Vacation`), отгул (`DayOff`), командировку (`BusinessTrip`), больничный (`SickLeave`),
    отпуск по уходу за ребёнком (`ParentalLeave`) или учебный отпуск (`StudyLeave`). Отсутствие привязывается
    к трудоустройству сотрудника.
</Info>

<Info>
    Создание, обновление и удаление отсутствия автоматически синхронизирует статус пользователя `Active` ↔
    `OnLeave`: если у пользователя есть текущее (действующее на данный момент) отсутствие — статус становится
    `OnLeave`, если текущих отсутствий не осталось — возвращается `Active`. `OnLeave` — информационный статус:
    вход и доступ к платформе он **не** блокирует. Переходы не затрагивают пользователей в статусах `Blocked`
    и `Terminated`. Синхронизация происходит в момент API-вызова: отсутствие с будущей датой начала не переключит
    статус само по себе в момент наступления этой даты.
</Info>

<Warning>
    Все эндпоинты модуля staff доступны **только** для школ сегмента `Corporate`. Для остальных сегментов запрос
    вернёт ошибку `403 Forbidden`.
</Warning>

## Список отсутствий

```
GET /saas/v2/staff/absence/list
```

Требуется аутентификация и право `StaffView`.

### Параметры запроса

<Info>
    Параметры-массивы передаются повторением параметра в строке запроса: `employmentIds=1&employmentIds=2`.
</Info>

#### Пагинация

<ParamField query="skip" type="integer" required={false}>
    Количество записей, которые нужно пропустить. По умолчанию `0`.
</ParamField>

<ParamField query="page" type="integer" required={false}>
    Номер страницы (альтернатива `skip`). Начинается с `1`.
</ParamField>

<ParamField query="take" type="integer" required={false}>
    Количество записей на странице. По умолчанию `10`.
</ParamField>

#### Фильтрация

<ParamField query="extIds" type="string[]" required={false}>
    Фильтр по внешним идентификаторам (`extId`) отсутствий. До 250 значений, каждое до 50 символов.
</ParamField>

<ParamField query="employmentIds" type="integer[]" required={false}>
    Фильтр по ID трудоустройств. До 250 значений.
</ParamField>

<ParamField query="positionIds" type="integer[]" required={false}>
    Фильтр по ID должностей. До 250 значений.
</ParamField>

<ParamField query="types" type="enum[]" required={false}>
    Фильтр по типам отсутствия. Возможные значения: `Absent`, `Vacation`, `DayOff`, `BusinessTrip`, `SickLeave`,
    `ParentalLeave`, `StudyLeave`. До 250 значений.
</ParamField>

<ParamField query="currentOnly" type="boolean" required={false}>
    Если `true` — вернуть только текущие (активные на данный момент) отсутствия.
</ParamField>

## Создать отсутствие

```
POST /saas/v2/staff/absence/create
```

Требуется аутентификация и право `StaffManage`.

### Параметры запроса

<Info>
    Трудоустройство указывается **ровно одним** из двух полей: `employmentId` или `employmentExtId`.
</Info>

<ParamField body="extId" type="string" required={false}>
    Внешний идентификатор отсутствия из системы клиента. От 1 до 50 символов, без `/` и пробелов. Должен быть
    уникален в рамках школы среди неудалённых записей — иначе вернётся ошибка `StaffAbsenceExtIdIsNotUniq`.
</ParamField>

<ParamField body="employmentId" type="integer" required={false}>
    ID трудоустройства сотрудника, к которому относится отсутствие. Обязателен, если не передан `employmentExtId`.
</ParamField>

<ParamField body="employmentExtId" type="string" required={false}>
    Внешний идентификатор (`extId`) трудоустройства. Альтернатива `employmentId` — обязателен, если не передан
    `employmentId`. Если трудоустройство не найдено в школе — вернётся ошибка `StaffEmploymentNotFound`.
</ParamField>

<ParamField body="type" type="enum" required>
    Тип отсутствия. Возможные значения: `Absent` (отсутствует), `Vacation` (отпуск), `DayOff` (отгул),
    `BusinessTrip` (командировка), `SickLeave` (больничный), `ParentalLeave` (отпуск по уходу за ребёнком),
    `StudyLeave` (учебный отпуск).
</ParamField>

<ParamField body="startAt" type="string" required>
    Дата начала отсутствия в формате ISO 8601. Должна быть не позже `finishAt`.
</ParamField>

<ParamField body="finishAt" type="string" required={false}>
    Дата окончания отсутствия в формате ISO 8601. Должна быть не раньше `startAt`.
</ParamField>

<ParamField body="note" type="string" required={false}>
    Комментарий к отсутствию. Максимум 500 символов. Автоматически обрезаются пробелы в начале и конце.
</ParamField>
