Этот гайд описывает типовой сценарий выгрузки сотрудников и оргструктуры из внешней системы (1С:ЗУП, CRM,
другая LMS) в Exode: порядок вызовов, маппинг типовых полей и куда передавать данные, для которых нет
прямого поля в API.

<Info>
  Ключевой принцип: все сущности связываются через **внешние идентификаторы** (`extId`) — GUID или коды из вашей
  системы. Вам не нужно хранить внутренние ID Exode: департаменты, должности и трудоустройства адресуются по
  `extId`, пользователи находятся по `extId` через [`user/find`](/ru/exode-api/school/user/find) и
  [`user/upsert`](/ru/exode-api/school/user/upsert).
</Info>

## Порядок синхронизации

<Steps>
<Step title="Департаменты">
  Создайте оргструктуру методами раздела [Департаменты](/ru/exode-api/school/staff/department). Передавайте
  `extId` (код подразделения из вашей системы) и `parentExtId` для иерархии.

  Родительский департамент должен существовать на момент создания дочернего — синхронизируйте дерево
  **сверху вниз** (или отсортируйте выгрузку так, чтобы родители шли раньше детей).

  ```json
  { "name": "Отдел продаж", "extId": "ПОДР-001", "parentExtId": "ПОДР-000" }
  ```
</Step>

<Step title="Должности">
  Создайте должности методами раздела [Должности](/ru/exode-api/school/staff/position) — `name` и `extId`.
  Если в вашей системе у должностей нет собственных кодов (передаётся только название, например
  `"job_position_name": "Менеджер"`) — сформируйте `extId` на своей стороне детерминированно, например
  транслитерацией или хешем названия, и переиспользуйте его в найме.
</Step>

<Step title="Сотрудники">
  Создавайте и обновляйте сотрудников одним вызовом [`user/upsert`](/ru/exode-api/school/user/upsert):
  пользователь ищется по логину / `tgId` / `extId`, при отсутствии — создаётся. Трудоустройство передаётся сразу
  в поле `staff` (`positionExtId` + `departmentExtId`).

  Если сотрудник числится в **нескольких подразделениях** — сделайте по одному
  [`staff/employment/hire`](/ru/exode-api/school/staff/employment#нанять-сотрудника) на каждое
  дополнительное подразделение: первое трудоустройство с `kind: "Main"`, остальные — `kind: "InternalSecondary"`.
  Дубли не создаются: повторная передача той же пары департамент + должность в `upsert` идемпотентна,
  а прямой повторный `hire` той же активной пары вернёт ошибку.
</Step>

<Step title="Руководители">
  Назначьте руководителей департаментов методом
  [`staff/department-manager/set`](/ru/exode-api/school/staff/department-manager): `departmentExtId` +
  `employmentExtId` (внешний идентификатор трудоустройства руководителя, который вы передали при найме).
  Основной руководитель помечается `isPrimary: true`.

  Если в вашей системе руководитель указывается у каждого сотрудника (например, `leader_external_ids`), а не у
  подразделения — назначайте руководителем департамента и/или сохраните ссылку на руководителя в кастомном поле
  сотрудника.
</Step>

<Step title="Остальные данные — кастомные поля">
  Для данных, которым нет поля в схеме пользователя (отчество, город, теги, произвольный статус из CRM,
  дополнительный email), создайте кастомные поля в [макете формы](/ru/exode-api/school/form-layout/create) и записывайте значения
  через [`custom-field/value/set-by-slug`](/ru/exode-api/school/custom-field/set) после апсерта пользователя.
</Step>

<Step title="Увольнения">
  При увольнении вызовите [`staff/employment/terminate`](/ru/exode-api/school/staff/employment#уволить-сотрудника)
  (по `extId` трудоустройства). Если доступ сотрудника нужно закрыть — дополнительно передайте `banned: true`
  в [`user/update`](/ru/exode-api/school/user/update) (все активные сессии будут завершены).

  Если ваша система отдаёт **полный список** сотрудников (а не изменения) — получите текущие активные
  трудоустройства через [`staff/employment/list`](/ru/exode-api/school/staff/employment), сравните с выгрузкой по
  `extId` и увольте отсутствующих.
</Step>
</Steps>

## Маппинг типовых полей

| Поле в HR-системе / 1С | Поле в Exode | Комментарий |
|---|---|---|
| Фамилия | `profile.lastName` | Максимум **15 символов**, только буквы и пробелы |
| Имя | `profile.firstName` | Максимум **15 символов**, только буквы и пробелы |
| Отчество | кастомное поле | Отдельного поля в профиле нет |
| Email | `email` | Является логином для входа |
| Телефон | `phone` | Международный формат, является логином |
| Логин из вашей системы | `domain` | Латиница, цифры и `_`, **точки недопустимы** — замените на `_` |
| GUID / внешний ID | `extId` | До 50 символов; **не логин** — авторизоваться по нему нельзя |
| Пол (`м`/`ж`) | `profile.sex` | Замаппьте на `Men` / `Women` (`Ufo` — не указан) |
| Дата рождения (ISO datetime) | `profile.bdate` | Формат `YYYY-MM-DD` — отбросьте время |
| Роль | `profile.role` | `Student` / `Tutor` / `Parent` |
| Пароль | `password` | Если `null` — Exode сгенерирует и отправит пароль автоматически |
| Название должности | `staff.positionExtId` | Предварительно создайте должность с `extId` |
| Код подразделения | `staff.departmentExtId` | Предварительно создайте департамент с `extId` |
| Несколько подразделений | несколько трудоустройств | `kind: Main` + `InternalSecondary` |
| Дата приёма на работу | `staff.startAt` | ISO 8601 |
| Статус («Работает» / «Уволен») | `status` | `Active` / `Terminated`; произвольные статусы — в кастомное поле |
| Руководители | `department-manager/set` | По `departmentExtId` + `employmentExtId` |
| Город, теги, прочее | кастомные поля | [`custom-field/value/set-by-slug`](/ru/exode-api/school/custom-field/set) |

## Ограничения, о которых стоит знать заранее

<Warning>
  - **`domain` не допускает точки** — только латинские буквы, цифры и `_`. Логины вида `i.ivanov.01011990`
    передавайте с заменой точек (`i_ivanov_01011990`) либо используйте для входа `email`.
  - **`firstName` и `lastName` — до 15 символов**, только буквы (латиница/кириллица) и пробелы. Длинные ФИО
    обрезайте на своей стороне.
  - **`extId` — до 50 символов**, без `/` и пробелов (используется в путях `ext/{extId}`).
  - **По `extId` и `tgId` нельзя авторизоваться** — поля для входа: `email`, `phone`, `domain`.
  - Модуль трудоустройств (`staff`) доступен только для **корпоративных школ**; для них поле `staff`
    обязательно при создании пользователя.
</Warning>

## Минимальный пример: сотрудник с трудоустройством

```bash cURL
curl --location --request PUT 'https://api.exode.biz/saas/v2/user/upsert' \
  --header 'Seller-Id: {{ sellerId }}' \
  --header 'School-Id: {{ schoolId }}' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer YOUR_TOKEN' \
  --data-raw '{
    "email": "ivanov@company.ru",
    "domain": "i_ivanov_01011990",
    "extId": "i.ivanov.01011990",
    "status": "Active",
    "profile": {
      "firstName": "Иван",
      "lastName": "Иванов",
      "bdate": "1990-01-01",
      "sex": "Men"
    },
    "staff": {
      "positionExtId": "manager",
      "departmentExtId": "ПОДР-001",
      "kind": "Main",
      "type": "FullTime",
      "startAt": "2020-03-01T00:00:00Z"
    }
  }'
```

После апсерта запишите нестандартные атрибуты в кастомные поля:

```bash cURL
curl --location 'https://api.exode.biz/saas/v2/form/custom-field/value/set-by-slug' \
  --header 'Seller-Id: {{ sellerId }}' \
  --header 'School-Id: {{ schoolId }}' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer YOUR_TOKEN' \
  --data-raw '{
    "userId": 1683,
    "layoutId": 12,
    "values": [
      { "slug": "middle_name", "value": "Иванович" },
      { "slug": "city", "value": "Ташкент" },
      { "slug": "crm_status", "value": "Работает" }
    ]
  }'
```
